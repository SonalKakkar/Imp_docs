**When you get a new MR / new build from developers, what is the first step?**

The first step is to understand the scope of changes from the MR or release notes — what functionality changed, impacted modules, and associated risks.

Then I ensure the build is deployable and stable, followed by smoke testing on critical flows.

If smoke passes, I decide whether to run sanity, targeted regression, or full regression based on change impact.

As a Test Lead, I also align resources, priorities, and automation execution.

**How will you perform performance testing?**

I start by identifying critical business transactions and expected load patterns.

Then I define scenarios like:, • Normal load, , Peak load, • Stress load, • Endurance testing

Tools:,  JMeter, k6, • Locust

I simulate concurrent users and measure response times, throughput, and error rates.

**🔥 4. Before saying performance testing is completed, what do you consider?**

I would ensure:

• Response times meet SLA

• No major errors under expected load

• System recovers after stress

• CPU / memory usage acceptable

• DB performance stable

• No memory leaks

• Bottlenecks identified and shared
